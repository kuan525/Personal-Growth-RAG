# 阶段执行手册

## 1. 文档目标

这份手册定义每个 phase 应该如何被严肃执行。

一个 phase 不是模糊的开发周期，而是一个受控单元，必须包含：

- goal
- scope
- deliverables
- validation
- failure criteria
- exit decision

## 2. Phase Start Checklist

开始任何 phase 前，必须确认：

- phase goal 清楚
- previous phase exit criteria 已通过
- required data 已准备
- non-goals 已列出
- validation plan 已准备
- rollback 或 stop condition 已知

## 3. Phase Work Template

每个 phase 应有工作记录：

```markdown
# Phase N: <Name>

## Goal

## Why now

## Scope

## Non-goals

## Deliverables

## Dependencies

## Implementation tasks

## Validation plan

## Pass criteria

## Fail criteria

## Known risks

## Exit decision
```

## 4. Task Breakdown Rules

每个 implementation task 必须回答：

1. 它让用户获得什么能力？
2. 可能影响哪些 file / module？
3. 什么输出证明它工作了？
4. 用什么 test 或 manual check 校验？
5. 明确不包含什么？

避免这种任务：

- improve retrieval
- add graph
- make memory better

使用这种任务：

- add BM25 retrieval and log retrieval_source for each result
- invalidate old chunks when document content_hash changes
- extract Project and Decision entities from decision logs with evidence_chunk_id

## 5. Validation Execution

每次 validation run 必须记录：

```text
phase
system version
config
question set version
input documents
run time
results
failure categories
next action
```

## 6. Exit Decision Types

每个 phase 结束时只能选择一种结论：

```text
pass_and_continue
partial_pass_fix_before_continue
fail_redesign_required
stop_scope_too_large
```

## 7. Quality Gate

出现以下情况，phase 不能通过：

- validation 没有执行
- 需要 citation 的地方没有 citation
- known failures 没有记录
- non-goals 被误实现并扰乱 scope
- next phase inputs 没准备好

## 8. Backlog Discipline

当出现新想法时，必须分类：

```text
current_phase_required
current_phase_optional
future_phase
reject
```

只有 `current_phase_required` 可以在当前 phase 通过前实现。

## 9. Documentation Rule

不要为了装饰创建文档。

每份文档必须属于以下类型之一：

- product requirement
- architecture decision
- data model
- implementation roadmap
- validation plan
- subsystem specification
- phase execution record

## 10. 当前执行建议

1. 创建真实 source sample set。
2. 写第一批 20 个 validation questions。

在 baseline 存在之前，不要实现 graph、memory 或 decision support。
