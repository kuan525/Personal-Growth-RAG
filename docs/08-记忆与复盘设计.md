# 记忆与复盘设计

## 1. 设计目标

Memory 的作用是把重复出现的文档、review 和 decision 变成长期个人上下文。

系统不能盲目“什么都记”。它应该只把有价值、有 source、可 review 的信息提升为 memory。

## 2. Memory 原则

1. Memory 必须有 source。
2. Memory 必须有 type。
3. Memory 在必要时必须有 time range。
4. Memory 必须可修正。
5. 临时 note 不能自动变成永久 fact。
6. Memory 应该改善未来 retrieval、review 或 decision。

## 3. Memory Types

初始 memory types：

```text
goal
preference
lesson
project_state
repeated_problem
decision_rationale
next_action
```

## 4. Review Sources

支持的 review-like inputs：

- weekly review
- project review
- milestone review
- decision review
- learning review

推荐 review 格式：

```markdown
# Review: <title>

- Date:
- Period:
- Project:

## What happened

## What was completed

## Problems

## Lessons

## Decisions

## Next actions
```

## 5. Decision Log Format

```markdown
# Decision: <title>

- Date:
- Project:
- Context:
- Options:
- Selected option:
- Reason:
- Evidence:
- Risks:
- Review date:
```

## 6. Memory Candidate Flow

1. Ingest review 或 decision note。
2. 抽取 candidate goals、lessons、problems、decisions、next actions。
3. 每个 candidate 关联 source chunk。
4. 标记 candidate status。
5. 用户 confirm、reject 或 edit。
6. confirmed memory 变为 active。

## 7. Memory Status

```text
candidate
confirmed
rejected
superseded
deleted
needs_review
```

## 8. 时间处理

Memory 应跟踪：

```text
created_at
updated_at
valid_from
valid_to
source_event_time
```

这样系统才能回答：

- 本月发生了什么变化？
- 哪些 problem 跨周反复出现？
- 哪些 goal 没有推进？
- 上个季度我做过什么 decision？

## 9. Memory Retrieval

query 涉及以下内容时，应检索 memory：

- personal preference
- project state
- repeated issue
- past decision
- next action
- periodic review
- planning

memory retrieval 应与 document retrieval 合并，而不是替代 document retrieval。

## 10. Review Summary 输出结构

一次 review summary 应包含：

1. period covered
2. main projects
3. completed work
4. repeated problems
5. important decisions
6. lessons
7. next actions
8. cited evidence

## 11. 验收标准

Memory and review 通过条件：

- review notes 能 ingestion
- candidate memories 有 sources
- time-based summaries 有 citation
- repeated problems 可检测
- 用户能 confirm / reject memories
- stale memories 可标记 needs_review

## 12. 失败条件

出现以下情况即失败：

- summary 泛泛而谈且无 citation
- memory 没有 source
- temporary thought 被当成 permanent fact
- 用户无法修正 memory
- time-based questions 无法回答
