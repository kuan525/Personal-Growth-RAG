# 实施路线图

## 1. 路线原则

路线图不按“功能看起来高级”排序，而按系统成熟度排序。

项目必须依次证明：

1. 系统能不能回答文档问题。
2. 系统能不能稳定找到正确 evidence。
3. 系统能不能安全维护变化中的资料。
4. 系统能不能用 personal graph 连接知识。
5. 系统能不能保留成长和项目历史。
6. 系统能不能基于 evidence 辅助决策。
7. 系统能不能在迭代中保持质量。

## 2. 阶段总览

| Phase | 名称 | 核心证明 | 退出标准 |
| --- | --- | --- | --- |
| 0 | 基础数据集准备 | 有真实数据和问题 | validation set 准备完成 |
| 1 | Local RAG MVP | 文档可以被问答 | citation QA 端到端跑通 |
| 2 | Retrieval Quality | 能找到正确 evidence | hybrid / rerank 优于 baseline |
| 3 | Structured Ingestion | 输入能被更好解析 | chunk 和 metadata 改善 |
| 4 | Incremental Knowledge Base | 数据变化可安全维护 | add/update/delete 生效 |
| 5 | Personal Graph MVP | 知识可以关联 | graph 改善真实问题 |
| 6 | Memory and Review | 经验可以沉淀 | review 变成 memory |
| 7 | Decision Support | 建议基于 evidence | decision 引用历史 |
| 8 | Operational Hardening | 质量可被保护 | regression workflow 存在 |

## 3. Phase 0：基础数据集准备

### 目标

在写代码之前，准备真实资料和真实问题。

### 交付物

- source sample folder
- 20 个 validation questions
- expected source annotations
- refusal questions
- 至少一份 review note
- 至少一份 decision note

### 退出标准

- 至少 5 个事实问题
- 至少 5 个概念问题
- 至少 4 个 multi-chunk 问题
- 至少 3 个 refusal 问题
- 至少 3 个 project / review / decision 问题

### 不做

- ingestion system
- UI
- graph
- memory

## 4. Phase 1：Local RAG MVP

### 目标

构建最小端到端文档问答系统，并支持 citation。

### 交付物

- file ingestion CLI
- PDF / Markdown / TXT parser
- chunking module
- embedding module
- vector index
- query CLI
- answer with citations
- query logs

### 退出标准

- 支持的文件类型都能 ingestion
- 真实文档上的 factual QA 可用
- answer 包含 citation
- refusal questions 不频繁 hallucinate
- query results 被记录

### 不做

- hybrid retrieval
- rerank
- graph
- 完整 incremental update
- Web UI

## 5. Phase 2：Retrieval Quality

### 目标

提升 retrieval 质量，让系统更稳定找到正确 evidence。

### 交付物

- BM25 或等价 sparse retrieval
- hybrid retrieval
- rerank
- retrieval_source tracking
- baseline comparison report

### 退出标准

- known-answer hit rate 提升
- difficult questions 改善
- citation quality 不退化
- latency 可接受
- failures 被分类

### 不做

- graph expansion
- complex agent planner
- automated tuning platform

## 6. Phase 3：Structured Ingestion

### 目标

提升 parsing 和 chunk quality。

### 交付物

- block-level intermediate representation
- PDF page metadata
- Markdown heading metadata
- structured chunking
- parse diagnostics

### 退出标准

- metadata 进入 citation
- difficult documents 表现改善
- chunk boundaries 更合理
- parse failures 可诊断

### 不做

- full OCR platform
- visual chart understanding
- 所有文件格式支持

## 7. Phase 4：Incremental Knowledge Base

### 目标

让系统能维护持续变化的资料。

### 交付物

- file scanner
- content hash tracking
- document status management
- chunk invalidation
- embedding update path
- index deletion / update path
- index status report

### 退出标准

- new document 无需 full rebuild 即可 searchable
- modified document 替代旧 active content
- deleted document 不再被 citation
- unchanged document 被跳过
- update logs 可查看

### 不做

- background daemon
- cloud sync
- multi-user corpus management

## 8. Phase 5：Personal Graph MVP

### 目标

连接高价值 personal knowledge entities。

### 交付物

- entity schema
- relation schema
- entity extraction
- relation extraction
- graph storage
- graph query
- graph-assisted retrieval

### 退出标准

- graph entries 有 evidence
- graph entries 可查看
- 错误 entries 可禁用或修正
- graph 至少改善 3 个真实问题

### 不做

- full GraphRAG
- graph database migration
- complex graph reasoning

## 9. Phase 6：Memory and Review

### 目标

把 review 和项目历史转成长期 memory。

### 交付物

- review note format
- decision note format
- goal / problem / lesson / next-action extraction
- memory candidate workflow
- time-based summaries

### 退出标准

- review 能 ingestion
- memory 有 source
- time-based summary 有 citation
- memory 可 confirmed / rejected / superseded

### 不做

- automatic life coach
- calendar integration
- emotional companion product

## 10. Phase 7：Decision Support

### 目标

基于个人历史辅助真实决策。

### 交付物

- decision question classifier
- decision context retrieval
- option comparison template
- recommendation generator
- risk and missing-info section
- decision record and review flow

### 退出标准

- recommendation 引用个人 evidence
- evidence 与 inference 分离
- risk 和 missing information 被列出
- decision 可后续 review

### 不做

- autonomous execution
- external system operation
- ungrounded life advice

## 11. Phase 8：Operational Hardening

### 目标

保护系统质量，避免迭代退化。

### 交付物

- stable evaluation set
- regression subset
- release checklist
- failure taxonomy
- baseline comparison process

### 退出标准

- 每次重要改动可 evaluation
- regression 可见
- failures 被分类
- release decision 基于 evidence

## 12. 依赖规则

- Phase 2 依赖 Phase 1。
- Phase 3 依赖 Phase 1/2 暴露出来的真实 failures。
- Phase 4 依赖稳定 document / chunk metadata。
- Phase 5 依赖 incremental update 稳定。
- Phase 6 依赖 graph 和 source traceability。
- Phase 7 依赖 memory 和 decision records。
- Phase 8 从 Phase 1 轻量开始，Phase 7 后正式化。

## 13. 当前最佳下一步

不要从 graph 或 decision support 开始。

当前应该做：

1. 准备 Phase 0 validation set。
2. 实现 Phase 1 Local RAG MVP。
3. 运行第一次 baseline evaluation。

只有 baseline failures 出现后，才知道 retrieval、parsing、graph、memory 该怎么优化。
