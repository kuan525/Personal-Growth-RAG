# 架构与数据模型

## 当前模块

```text
src/personal_growth_rag/
├── api/v1/          # HTTP routes + contracts
├── application/     # 业务用例和主数据流
├── domain/          # 轻量业务概念、prompt、稳定规则
├── infrastructure/  # DB、FAISS、DashScope、DeepSeek、parser、chunker
├── core/            # 配置、日志、错误、路径、常量
├── cli/             # 本地命令入口
├── utils/           # hash、id、time 等工具
└── main.py          # FastAPI 入口
```

## 分层职责

`api` 只提供 HTTP 入口，不承载核心逻辑；`cli` 和 `api` 都调用 application use case。`application` 负责主数据流编排，例如 ingest document、search chunks、answer question。`domain` 保持轻量，放稳定概念、prompt 和少量规则。`infrastructure` 负责 SQLite、FAISS、DashScope、DeepSeek、pdfminer 等具体实现。`core` 只放全局基础能力。


## 运行时依赖方向

```text
api/v1/routes 或 cli
  -> application/* use case
    -> domain dataclass / prompt
    -> infrastructure/db repositories
    -> infrastructure providers / vectorstores
```

约束：API contract、domain data、SQLAlchemy model 三者不要混用。API contract 面向外部调用方，domain data 面向内部用例，SQLAlchemy model 只代表数据库行。

## 当前存储

```text
data/
├── app.db        # SQLite metadata database
├── uploads/      # 原始上传文件副本
├── chunks/       # chunk JSON 调试文件
└── indexes/      # FAISS index 文件
```

`data/` 是本地运行数据，不进入 Git。

## 当前数据表

### documents

文档级 metadata 和处理状态。

核心字段：

```text
id
source_name
file_type
status
content_hash
stored_path
chunk_path
chunk_count
error_message
created_at
updated_at
```

当前状态：

```text
processing
active
failed
```

后续增量更新会扩展：

```text
deleted
superseded
```

### chunks

可检索文本单元。

核心字段：

```text
id
document_id
chunk_order
text
content_hash
created_at
```

约束：

- `document_id` 关联 `documents.id`。
- `document_id + chunk_order` 唯一。

### embeddings

chunk 到 FAISS index position 的映射。

核心字段：

```text
id
chunk_id
model_name
vector_dim
index_position
created_at
```

当前 FAISS 是 SQLite metadata 的派生索引。后续删除、更新、compact 应优先通过 rebuild index 保证一致性。

```text

retrieval_result -> chunk -> document

```
