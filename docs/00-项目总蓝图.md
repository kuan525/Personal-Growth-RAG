# 项目总蓝图

## 定位

Personal Growth RAG 是一个本地优先的个人知识系统。它从个人文档、笔记、项目记录、复盘和决策材料中提取 evidence，逐步形成可检索、可引用、可更新、可复盘的知识库。

它不是普通文档问答 demo，长期目标是服务个人项目推进、学习复盘和关键决策。

## 核心原则

1. **Evidence before opinion**：回答和建议必须基于可追溯来源。
2. **Traceability before intelligence**：先让 evidence、retrieval、answer 可追踪，再追求更聪明的生成能力。
3. **Local-first before platform**：数据默认保存在本地，先服务单用户真实场景。
4. **Incremental by design**：文档、chunk、embedding、index 后续都要支持新增、更新、删除。
5. **Small steps before big architecture**：先打通最小 RAG 闭环，再扩展 graph、memory、decision support。

## 能力分层

```text
Sources
  -> Ingestion
  -> Chunking
  -> Metadata Store
  -> Embedding / Indexing
  -> Retrieval
  -> Answer with Citations

Future:
  -> Question Trace
  -> Incremental Update
  -> Evaluation / Regression
  -> Personal Graph
  -> Memory / Review
  -> Decision Support
```

## 当前阶段

当前已完成最小 RAG MVP：

- 文档上传和本地目录批量导入。
- `.txt` / `.md` / `.pdf` 解析。
- 基础 chunking。
- SQLite metadata 落库。
- DashScope embedding。
- FAISS 本地向量索引。
- `/v1/search` 语义检索。
- `/v1/questions` 基于 retrieval 的 LLM 回答与 citations。

下一步优先实现 Question Trace MVP，让每次 question、retrieval results、answer、citations 都能落库、回放、评估和复盘。

## 明确不做

当前不优先做：

- 多用户和权限系统
- SaaS 化
- Web UI / TUI
- 完整 OCR
- 复杂 agent workflow
- 一开始就做完整 GraphRAG
