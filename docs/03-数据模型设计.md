# 数据模型设计

## 1. 数据模型目标

数据模型必须支撑一个长期成长的个人知识系统，而不只是支撑一次 query。

它需要同时服务：

- incremental update
- citation traceability
- personal graph
- long-term memory
- decision review
- evaluation / regression

核心追溯链路：

```text
answer -> citation -> chunk -> document
relation -> evidence chunk -> document
memory -> source chunk -> document
decision -> evidence -> outcome review
```

## 7. 数据规则

1. chunk 不能脱离 document 存在。
2. citation 不能脱离 chunk 存在。
3. relation 不能没有 evidence。
4. memory 没有 source evidence 时不能成为 confirmed。
5. decision recommendation 必须记录 retrieved evidence。
6. deleted / superseded chunk 默认不能参与 retrieval。
7. eval result 必须保留当时使用的 system configuration。
