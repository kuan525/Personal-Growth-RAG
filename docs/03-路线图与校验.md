# 路线图与校验

## 当前进度

已完成：

1. FastAPI 服务骨架与 `/v1/health`。
2. `POST /v1/documents`。
3. `.txt` / `.md` / `.pdf` 解析。
4. 基础 chunking。
5. SQLite + SQLAlchemy metadata 落库。
6. DashScope embedding。
7. FAISS 本地向量索引。
8. `GET /v1/documents` 和 `GET /v1/documents/{document_id}`。
9. `POST /v1/search`。
10. `POST /v1/questions`，完成 answer with citations 的最小 RAG 闭环。
11. 项目结构重构为 `api / application / domain / infrastructure / core`。


## 当前重构完成标准

本次结构重构已完成以下约束：

- 包名统一为 `personal_growth_rag`。
- 公开 API 统一为 `/v1/*`。
- API 请求/响应契约集中在 `api/v1/contracts`。
- API 和 CLI 共享 application use case。
- 成功和失败响应统一为 `success / error_code / message / data / request_id` envelope。
- `docs/` 作为正式项目文档进入 Git。

## 下一阶段：Question Trace MVP

目标：每次问答都可以回放、评估、复盘。

需要实现：

1. `query_runs` 表记录 question、top_k、status、error_message。
2. `retrieval_results` 表记录每次检索命中的 chunk、rank、score。
3. `answers` 表记录 answer、model_name、max_tokens。
4. `citations` 表记录 answer 到 chunk 的引用关系。
5. `/v1/questions` 返回 `query_run_id`。
6. 后续增加 `GET /v1/question-runs/{query_run_id}` 回放接口。

## 后续路线

| 阶段 | 目标 | 核心产物 |
| --- | --- | --- |
| Retrieval Quality | 提升 evidence 命中率 | hybrid retrieval、rerank、baseline 对比 |
| Structured Ingestion | 提升解析和 chunk 质量 | heading/page metadata、block representation |
| Incremental Update | 安全处理资料变化 | new/modified/deleted/unchanged flow、index rebuild |
| Evaluation | 防止质量回退 | validation set、failure log、regression run |
| Personal Graph | 连接项目、技能、目标、问题、决策 | entities、relations、evidence links |
| Memory & Review | 沉淀长期个人上下文 | review ingestion、memory candidates |
| Decision Support | 基于 evidence 辅助选择 | option compare、risk、missing info、decision record |

## 校验原则

每个阶段完成标准不是“代码写完”，而是在真实数据上证明能力可用。

长期保留以下校验资产：

- validation questions
- expected source annotations
- unsupported / refusal questions
- baseline run results
- failure log
