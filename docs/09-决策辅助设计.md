# 决策辅助设计

## 1. 设计目标

Decision support 是这个系统最高价值的一层。它的目标不是给泛泛建议，而是基于你的个人历史、evidence、goal、risk 和 lesson，辅助你做更清醒的选择。

系统不能像普通聊天助手一样输出没有来源的建议。

## 2. Decision Support 原则

一个 decision answer 必须区分：

- source facts
- historical patterns
- model inference
- risks
- missing information
- recommended next action

## 3. 支持的 Decision Types

初始类型：

```text
project_priority
learning_direction
implementation_tradeoff
phase_exit_decision
focus_planning
retrospective_adjustment
```

## 4. Decision Query Flow

1. 接收 decision question。
2. 识别 decision type。
3. 提取 object、constraints、time range、goal。
4. 检索相关 documents、graph relations、memories、previous decisions。
5. 如果信息不足，提出 clarification。
6. 比较 options。
7. 基于 evidence 给出 recommendation。
8. 列出 risks 和 missing information。
9. 给出 smallest next action。
10. 如果用户接受，保存 decision record。

## 5. 必须输出的结构

每次 decision answer 应包含：

```text
1. Decision question
2. Current context
3. Relevant evidence
4. Options
5. Recommendation
6. Why this option
7. Risks and counter-evidence
8. Missing information
9. Smallest next action
10. Sources
```

## 6. Evidence Rules

Decision support 必须引用：

- related project notes
- related reviews
- related prior decisions
- relevant memories
- relevant document chunks

如果 evidence 不足，系统必须明确说明。

## 7. Recommendation Rules

只有满足以下条件，系统才允许给明确 recommendation：

- 至少存在一个 relevant source
- 当前 goal 可识别
- options 可以比较
- uncertainty 被明确说明

否则，系统应该提出 clarification，或只给 limited analysis。

## 8. Decision Record

保存的 decision 应包含：

```text
decision_id
title
context
options
selected_option
reason
evidence_chunk_ids
decision_time
review_time
outcome
status
```

## 9. Decision Review

后续 review 应回答：

- 这个 decision 是否被执行？
- 结果发生了什么？
- 什么有效？
- 什么失败？
- 应该沉淀什么 lesson？
- 这个 decision 应该继续、反转还是更新？

## 10. 验收标准

Decision support 通过条件：

- recommendation 引用 personal evidence
- options 被比较
- evidence 和 inference 分离
- risks 被列出
- missing information 被列出
- decision 可记录并后续 review

## 11. 失败条件

出现以下情况即失败：

- 输出是 generic advice
- 没有 sources
- uncertainty 被隐藏
- recommendation 忽略用户历史
- 系统从不 clarification
- decision 无法后续 review
